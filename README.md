# A股每日盘后报告（网页版）—— 部署指南

这次的方案更简单：**不用配置任何邮箱**，程序每天自动把报告生成到一个网页上，
你收藏一个固定网址，每天点开看最新的就行。

**报告内容包括：**
- 大盘资金流概览
- 今日行业板块资金流排名（找热点板块）
- 🔥 持续吸金行业板块（连续多日被资金关注，不是一日游）
- 今日概念板块资金流排名（找热点题材）
- 🔥 持续吸金概念板块
- 个股资金净流入排行
- 今日涨停板池（市场情绪）
- 今日龙虎榜明细（主力/游资动向）

**先说清楚一点**：这套系统只做"数据搬运工"，不会告诉你"明天哪个股票一定涨"。
资金流入多只代表"被关注/被买入"，不代表接下来会继续涨，仅供参考。

---

## 需要上传的文件（一共3个 + 1个空文件夹）

```
main.py                              ← 主程序
requirements.txt                     ← 依赖清单
.github/workflows/daily_report.yml   ← 定时任务配置
docs/index.html                      ← 占位网页（首次运行前的提示页）
```

---

## 第一步：注册 GitHub 账号（如果还没有）

打开 https://github.com 注册一个账号，免费。

## 第二步：新建一个仓库

1. 登录后点右上角 "+" → "New repository"
2. 仓库名随便起，比如 `my-stock-report`
3. **这次选 Public（公开）**，不要选 Private —— 因为免费版 GitHub Pages（生成网页功能）在私有仓库上有使用限制，公开仓库没有这个问题。
   （仓库公开不代表你的邮箱密码会暴露，我们这次方案压根不涉及任何邮箱账号密码，所以完全没有隐私顾虑）
4. 点 "Create repository"

## 第三步：上传文件

在仓库页面点 "Add file" → "Upload files"，把下面这几个文件依次传上去：

1. `main.py`
2. `requirements.txt`
3. `.github/workflows/daily_report.yml` —— **网页拖拽经常会丢失这种带文件夹的文件**，
   如果拖拽上传后发现没有生成 `.github` 文件夹，改用下面这个更保险的方法：
   - 点 "Add file" → "Create new file"
   - 文件名那一栏直接输入完整路径：`.github/workflows/daily_report.yml`
   - 把内容粘贴到下方文本框，点 "Commit changes"
4. `docs/index.html` 同理，如果拖拽上传后 `docs` 文件夹没出现，就用 "Create new file"，
   文件名输入 `docs/index.html`，粘贴内容后提交

上传完，回到仓库主页确认能看到 `main.py`、`requirements.txt`、`.github` 文件夹、`docs` 文件夹这四样都在。

## 第四步：给自动化流程开写权限

因为这次程序需要把生成的网页自动提交回仓库，需要额外开一个权限：

1. 仓库页面 → "Settings"
2. 左侧菜单找 "Actions" → "General"
3. 拉到最下面 "Workflow permissions" 这一块
4. 选择 **"Read and write permissions"**
5. 点 "Save"

## 第五步：开启 GitHub Pages（把 docs 文件夹变成一个网站）

1. 还是在 "Settings" 页面，左侧菜单找 "Pages"
2. "Source" 选择 **"Deploy from a branch"**
3. "Branch" 选择 **"main"**，右边文件夹选择 **"/docs"**
4. 点 "Save"
5. 保存后页面会显示一行提示，类似：
   `Your site is live at https://你的用户名.github.io/仓库名/`
   （这个网址可能要等1-2分钟才生效，刷新页面看）
6. **把这个网址收藏起来**，以后每天就打开这个网址看报告

## 第六步：手动运行一次测试

1. 点仓库上方 "Actions" 标签
2. 左侧应该能看到 "A股每日盘后报告" 这个工作流，点进去
3. 右侧有个 "Run workflow" 按钮，点开下拉框，再点绿色的 "Run workflow" 确认
4. 等1-2分钟，刷新页面，绿色勾 ✅ 说明成功；红色叉 ❌ 点进去看日志报错
5. 成功后，打开第五步保存的那个网址，应该就能看到今天的报告了（如果网页还是"报告还没生成"那句占位文字，说明网页缓存没刷新，等一两分钟再刷新试试，或者用无痕模式打开）

测试成功后就不用管了，之后每个交易日北京时间 16:35 左右会自动更新这个网页。

## 常见问题

**Q: Actions 左侧列表里找不到 "A股每日盘后报告" 这个工作流？**
A: 说明 `.github/workflows/daily_report.yml` 没有正确上传，用第三步里"Create new file"的方法重新建一次。

**Q: 运行后是红叉❌，报错跟 git push 权限有关？**
A: 说明第四步"Read and write permissions"没设置成功，回去检查一下并保存。

**Q: 打开网址显示 404 或者打不开？**
A: 确认第五步 GitHub Pages 已经保存成功，且等了1-2分钟。也确认仓库是 Public 不是 Private。

**Q: 报告里某一项显示"抓取失败"？**
A: 数据源东方财富网偶尔会临时限流，正常现象，一般第二天自动恢复。

**Q: 想改成每天都跑（包括周末）或者改时间点？**
A: 打开 `.github/workflows/daily_report.yml`，改这一行：`cron: "35 8 * * 1-5"`
（前面 `35 8` 是UTC时间8点35分，对应北京时间16:35；后面 `1-5` 是周一到周五）。
不熟悉的话把需求发给我，我帮你改。

**Q: 想加更多数据，比如自选股实时资金流？**
A: 完全可以，需求告诉我，我在 `main.py` 里加对应代码。
